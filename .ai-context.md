# .ai-context — Fantasy Bakery showroom

Paste into **Cursor Rules** and **Antigravity** session / system instructions. Update when the store or API changes.

---

## Stack

- **Backend:** FastAPI (`app.py`). Showroom product JSON: **`GET /api/showcase/product`**.
- **Frontend:** **`frontend/`** — Vite + React + TypeScript + **Zustand** + **`@react-three/fiber`** (`Canvas` only in Architect-owned shell).
- **3D assets:** **`frontend/public/models/`** → URL path **`/models/...`**.

---

## Split

| Who | Scope |
|-----|--------|
| **Architect (Cursor)** | React app, Zustand **definition**, 2D UI (price, WhatsApp), FastAPI integration, **`ShowroomCanvas`** = `<Canvas>` wrapper + defaults — **no** meshes/lights/loaders inside. |
| **Engine (Antigravity)** | Everything **inside** the Canvas: scene graph, GLB, textures, lighting, `useFrame`. **Only** mutates scene state via **`setSceneBridge`**. |

---

## Zustand (`frontend/src/store/showcaseStore.ts`)

**Engine → UI (write via `setSceneBridge` only):**

- `meshLoaded: boolean`
- `rotationDeg: number`
- `facingCamera: boolean`

**API → UI (Architect):** `hydrateProductFromApi` from `/api/showcase/product` fields: `name`, `price_kes`, `whatsapp_e164`, `whatsapp_message`.

## Dev

- API: `uvicorn app:app --reload` on **8000**.
- UI: `cd frontend && npm run dev` — proxies **`/api`** to 8000.
